Dữ liệu time-series cần được giữ đúng thứ tự.

Dataset NAB ambient temperature mô phỏng telemetry nhiệt độ. Feature của model phụ thuộc lịch sử gần đây, nên timestamp, sort order và split theo thời gian là phần quan trọng nhất.

Feature engineering

hour

Giúp model học pattern ngày đêm hoặc giờ vận hành.

dayofweek

Hữu ích khi thiết bị có hành vi khác giữa ngày làm việc và cuối tuần.

rolling_mean_12

Mức nền ngắn hạn, khoảng 1 giờ nếu sampling 5 phút.

rolling_std_12

Mức dao động gần đây. Rất thấp có thể gợi ý sensor bị kẹt.

rolling_mean_36

Mức nền dài hơn, khoảng 3 giờ với dữ liệu hiện tại.

delta_1 / delta_3

Bắt thay đổi đột ngột so với điểm trước hoặc vài điểm trước.

zscore_rolling

Đo mức lệch khỏi pattern gần đây.

is_stuck_candidate

Cờ cảnh báo sensor giữ giá trị gần như không đổi.

value

Giá trị hiện tại vẫn là tín hiệu trực tiếp quan trọng.

Không random split time-series Random split làm model nhìn thấy pattern tương lai, khiến metric đẹp giả. Lab dùng split theo thời gian để mô phỏng train trên quá khứ và test trên tương lai.
Feature phải giống giữa train và API API gọi lại add_time_features(). Đây là điểm tốt, nhưng phải gửi đủ history để rolling feature đáng tin.

Nếu phát triển tiếp

  • Thêm device_id để tính feature riêng theo từng sensor.
  • Kiểm tra missing timestamp, duplicate timestamp và gap dữ liệu.
  • Dùng sin/cos cho hour để biểu diễn chu kỳ 24 giờ tốt hơn.
  • Thêm rolling median, quantile và rate-of-change theo phút.
