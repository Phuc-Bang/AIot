Lab 3 phát hiện bất thường hay dự báo tương lai?

Lab 3 phát hiện bất thường. Model trả lời điểm hiện tại có khác pattern bình thường không, không dự báo nhiệt độ trong tương lai.

Vì sao không random split?

Random split trộn quá khứ và tương lai, gây data leakage. Với time-series, phải train trên quá khứ và test trên tương lai.

If-else threshold có đủ không?

Đủ cho rule vật lý đơn giản, nhưng không đủ khi bất thường phụ thuộc context như rolling mean, delta, giờ trong ngày hoặc sensor stuck.

Anomaly detection hơn if-else ở đâu?

Anomaly detection học pattern bình thường từ dữ liệu và phát hiện tổ hợp tín hiệu bất thường, ví dụ giá trị chưa vượt ngưỡng nhưng tăng quá nhanh so với nền gần đây.

Precision thấp nguy hiểm như nào?

Precision thấp tạo nhiều false alert, làm operator mất niềm tin, dashboard nhiễu và tăng chi phí xử lý.

Recall thấp nguy hiểm như nào?

Recall thấp nghĩa là bỏ sót anomaly thật. Với hệ thống liên quan an toàn, missed alert có thể gây thiệt hại lớn hơn false alert.

False alert và missed alert cái nào nguy hiểm hơn?

Phụ thuộc domain. Kho lạnh vaccine, điện, máy công nghiệp thường ưu tiên giảm missed alert. Hệ thống ít quan trọng có thể ưu tiên giảm false alert để tránh spam.

Test notebook khác deploy API như nào?

Notebook đánh giá model offline trên tập test có label, sinh metric và chart. API phục vụ request mới, thường không có label ngay, và phải giữ response contract ổn định.

Vì sao có anomaly_score nhưng vẫn cần severity?

Score là tín hiệu model. Severity là mức nghiêm trọng vận hành, cần thêm ngữ cảnh như device criticality, duration và rule an toàn.

Tại sao API cần history chứ không chỉ một điểm?

Feature rolling như rolling_mean_36, delta_1 và zscore_rolling cần ngữ cảnh gần đây để tính đúng.

Kết quả model có phải xác suất không?

Không. anomaly_score là điểm bất thường đã chuẩn hóa để dễ diễn giải, không phải xác suất thiết bị hỏng.

Khi nào nên retrain model?

Khi đổi feature, đổi dataset, thêm sensor mới, data drift, feedback label thay đổi hoặc alert quá nhiều/bỏ sót quá nhiều.

Điểm cần nhớ Training và API hiện chưa dùng cùng một cách normalize score. Đây là câu hỏi kỹ thuật quan trọng khi review lab.
