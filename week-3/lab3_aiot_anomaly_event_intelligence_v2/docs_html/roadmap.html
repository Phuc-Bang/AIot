Lab 3 - Anomaly Detection & Event Intelligence for AIoT

Từ lab thành hệ thống AIoT thật.

Không nên bắt đầu bằng model phức tạp. Ưu tiên đầu tiên là data reliability, scoring contract, event policy và khả năng vận hành.

Anomaly Score Severity Decision FastAPI

Basic

  • Đồng bộ scoring giữa train và API.
  • Lưu threshold, score min/max và feature list cùng model.
  • Validate history length tối thiểu.
  • Trả HTTP status code đúng khi model chưa load.
  • Viết test schema cho /detect-anomaly.

Good

  • Thêm validation split theo thời gian để tune threshold.
  • Tách event rule và cooldown chống alert fatigue.
  • Ghi event vào SQLite hoặc PostgreSQL.
  • Tạo dashboard quan sát score, event và incident.
  • Hỗ trợ nhiều device_id và feature theo từng sensor.

Advanced

  • Event aggregation thành incident lifecycle.
  • Docker hóa API, database và dashboard.
  • Monitoring data drift và model drift.
  • Kết nối MQTT, ESP32 hoặc sensor thật.
  • Model registry, versioning và retraining pipeline.
