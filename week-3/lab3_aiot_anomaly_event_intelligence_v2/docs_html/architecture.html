Lab 3 - Anomaly Detection & Event Intelligence for AIoT

Architecture pipeline

Kiến trúc Lab 3 tách rõ hai lớp: lớp ML tạo anomaly score và lớp backend/event quyết định cách phản ứng. Đây là cách thiết kế thực tế hơn so với việc để model trực tiếp quyết định hành động.

Sơ đồ Mermaid

flowchart LR A["Telemetry"] --> B["Feature engineering"] B --> C["Anomaly model"] C --> D["anomaly_score"] D --> E["Event layer"] E --> F["Decision layer"] F --> G["API"]
Ghi chú Nếu mở file offline và Mermaid không render, nội dung pipeline vẫn là: telemetry -> feature engineering -> anomaly model -> anomaly_score -> event layer -> decision layer -> API.

Offline flow

Tải dữ liệu
Chạy src/download_data.py để tải NAB hoặc dùng sample.
Tạo feature
src/utils.py tạo rolling mean, rolling std, delta và z-score.
Train model
src/train_anomaly.py train Isolation Forest và autoencoder demo.
Đánh giá
Metrics, prediction CSV và event log được ghi vào outputs/.

Online flow

Client gửi history
API nhận danh sách telemetry gần nhất.
API tạo feature
Dùng lại add_time_features() để giữ logic gần training.
Score điểm mới nhất
Model chỉ score dòng cuối cùng trong cửa sổ history.
Trả event JSON
Response có model_output, event, api_check.

API output

{
  "model_output": { "anomaly_score": 0.61, "is_anomaly": true },
  "event": { "severity": "MEDIUM", "decision": "CREATE_WARNING_EVENT" },
  "api_check": { "latency_ms": 12.3 }
}
Nhận xét kỹ thuật Training đang normalize score bằng min-max trên tập test, còn API dùng sigmoid demo. Đây là điểm cần đồng bộ nếu muốn triển khai thật.
Thiết kế đúng hướng Event layer tách khỏi model giúp hệ thống thêm policy, cooldown, audit và safety rule mà không phải retrain model.
